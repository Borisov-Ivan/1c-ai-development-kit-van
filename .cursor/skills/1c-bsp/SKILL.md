# 1C БСП - Integration Toolkit

Инструменты для интеграции с Библиотекой Стандартных Подсистем (БСП). Регистрация обработок, добавление команд, готовые паттерны.

## 🎯 Quick Start

### Зарегистрировать обработку в БСП
```bash
# Добавить СведенияОВнешнейОбработке() в модуль объекта
1c-bsp registration add DataProcessors/МояОбработка/Ext/ObjectModule.bsl "Моя обработка" "Обработка данных"
```

### Добавить команду БСП
```bash
# Добавить команду в СведенияОВнешнейОбработке()
1c-bsp command add DataProcessors/МояОбработка/Ext/ObjectModule.bsl "Обработать" "ОбработатьДанные"
```

## 📦 Включенные Skills

> **Покрытие подскиллов:** registration и command охватывают только подсистему «Дополнительные отчеты и обработки».
> Для остальных подсистем БСП (ДлительныеОперации, РаботаСФайлами, ВерсионированиеОбъектов, УправлениеПечатью, РаботаСПочтой и др.) — используй `user-1c-ssl-ssl_search` и `1c-bsp/patterns/SKILL.md`.

### 1. registration - Регистрация в БСП
**Путь**: `1c-bsp/registration/SKILL.md`
- Добавляет функцию СведенияОВнешнейОбработке()
- Настраивает метаданные для подсистемы "Дополнительные отчеты и обработки"
- Определяет назначение и вид обработки

### 2. command - Команды БСП
**Путь**: `1c-bsp/command/SKILL.md`
- Добавляет команду в СведенияОВнешнейОбработке()
- Генерирует обработчик команды
- Настраивает видимость и доступность

### 3. patterns - Паттерны работы с БСП
**Путь**: `1c-bsp/patterns/SKILL.md`
- Пользователи и права
- Работа с файлами
- Печатные формы
- Фоновые задания
- Версионирование объектов
- Отправка email

## 🔄 Workflow

```yaml
Интеграция обработки с БСП:
  1. Зарегистрировать: 1c-bsp registration
  2. Добавить команды: 1c-bsp command
  3. Использовать паттерны: 1c-bsp patterns
  4. Протестировать в подсистеме

Использование паттернов:
  1. Изучить: 1c-bsp patterns [topic]
  2. Применить готовый код
  3. Адаптировать под задачу
```
