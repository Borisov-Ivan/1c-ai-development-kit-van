# openspec-docs

Генерация проектных документов на основе спецификаций OpenSpec.

## Принцип

Спецификации в `openspec/specs/` и контекст проекта в `openspec/project.md` — единственный источник. Документы генерируются без додумывания: если информации нет в спеках — она не попадает в документ.

## Поддерживаемые типы документов

| Тип | Команда | Промпт | Аудитория |
|-----|---------|--------|-----------|
| Функциональные требования | `/docs:functional` | `prompts/functional.md` | Бизнес-заказчик, пользователи |
| Технический проект | `/docs:technical` | `prompts/technical.md` | Разработчики (будет добавлен) |

## Алгоритм

1. **Определить тип документа** из запроса пользователя.

2. **Прочитать контекст проекта:**
   ```
   openspec/project.md
   ```

3. **Прочитать все спецификации:**
   ```
   openspec/specs/*/spec.md
   ```

4. **Прочитать промпт** из `prompts/<тип>.md`.

5. **Применить промпт** к собранным данным — сгенерировать документ.

6. **Показать результат** пользователю и спросить, нужны ли корректировки.

7. **Записать** в `docs/<название документа>.md` после подтверждения.

## Фильтрация спеков по типу документа

- **Функциональные требования:** используются спеки, Purpose которых описывает пользовательский или административный опыт. Чисто технические спеки (транспорт, хеширование, внутренние перехваты) не включаются.
- **Технический проект:** используются все спеки без исключения.

Определение «внутренний спек» — по Purpose: если в описании нет упоминания пользователя, администратора, формы или интерфейса, спек считается внутренним.
