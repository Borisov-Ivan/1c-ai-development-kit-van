# Скилл вендорских стандартов для ИИ-агентов

## Состояние файла standards-1c.md

Файл ~9800 строк, содержит текст, извлечённый из ИТС (v8std). Качество: OCR-артефакты ("о бщие", "бъектов"), сломанные таблицы, фрагменты кода без форматирования, блоки `## ВЫБРАТЬ` вместо примеров внутри разделов. Содержимое покрывает все 13 разделов ИТС:

1. Создание и изменение объектов метаданных
2. Реализация обработки данных
3. Соглашения при написании кода
4. Клиент-серверное взаимодействие
5. Общие вопросы безопасности
6. Настройка прав доступа к данным
7. Реализация обмена данными
8. Разработка и использование библиотек
9. Требования по локализации
10. Проектирование интерфейсов для 8.3
11. Разработка пользовательских интерфейсов (управляемое)
12. Проектирование интерфейсов для 8.2
13. Разработка пользовательских интерфейсов (обычное приложение)

## Анализ релевантности для агентов

### УДАЛИТЬ полностью (~2500-3000 строк, ~30% файла)

**Раздел «Обычное приложение» (строки ~9400-9800):** Интерфейс "Полный", имена элементов управления, размеры формы, гиперссылки в диалогах, флаги автовыбора, стили обычного приложения. Агенты работают только с управляемым приложением; обычное приложение — legacy, не поддерживается в расширениях.

**Раздел «Проектирование интерфейсов для 8.2» (~500 строк):** Устаревший раздел; платформа 8.2 не актуальна.

**Подробности UI/UX, специфичные для дизайнера (не программиста):**

- Тумблер (строка ~6510) — правила использования конкретного элемента
- Подсказки на форме (строка ~6520) — текстовые подсказки, расширенные подсказки
- Шрифты (строка ~6554) — MS Core Fonts, стилевые шрифты
- Поле "Дата" в списках (строка ~7563) — ширина колонки 9 пунктов
- Пользовательские представления объектов — синонимы, представления списков (строка ~6575)
- Окна на старте (строка ~8587)
- Подробные скриншоты/описания UI для вкладок, группировки элементов
- Подробности шаблонов формы документа (конкретные отступы, размеры полей)

**Раздел «Нумерация редакций и версий» (строки ~687-720):** Правила нумерации версий конфигурации (X.Y.Z.W). Не относится к написанию кода.

**Раздел «Требования к установке и обновлению» (строки ~647-686):** Процесс установки и обновления. Не код.

**Раздел «Заполнение свойств конфигурации информацией о выпуске» (строки ~720-724):** Свойства конфигурации при выпуске. Не код.

**Раздел «Общие сведения о выпуске конфигураций» (строки ~687-693):** Организационные процессы. Не код.

**Подробности реализации обмена данными:** Планы обмена, EnterpriseData, сериализация — агенты не реализуют обмен. Оставить только принцип ОбменДанными.Загрузка (уже в обработчиках событий).

### СОКРАТИТЬ значительно (~2000 строк)

**Примеры запросов (строки ~1600-2100):** Сейчас ~500 строк с многочисленными `## ВЫБРАТЬ`, `## ИЗ`, `## УПОРЯДОЧИТЬ ПО` — это сломанное форматирование. Оставить правила + 1-2 примера на каждый, остальные убрать.

**Права доступа (строки ~4400-4600):** Детальное описание ролей ПолныеПрава, АдминистраторСистемы, всех 12 подролей (ЗапускТонкогоКлиента, ВыводНаПринтерФайлБуферОбмена и т.д.). Агентам достаточно принципов: стандартные роли, проверка прав через ПравоДоступа (не РольДоступна), привилегированный режим. Детали подролей — убрать.

**Информирование пользователя (строки ~8400-8600):** Подробные правила UX-текстов (3 вопроса в ошибке, ширина оповещений 36 знаков, кнопки по умолчанию). Агентам достаточно: не Сообщить, а БСП-метод; сообщения привязать к полю; в транзакции не выводить сообщения.

**Динамические списки (строки ~7400-7600):** Подробные рекомендации по оптимизации динамических списков. Оставить 5-7 ключевых правил, убрать подробные примеры с ПриПолученииДанныхНаСервере.

**Локализация (строки ~5500-5600):** Подробные примеры с СтрШаблон/НСтр/ПолноеИмя. Оставить принцип (НСтр обязательна) + 1 пример.

## Шаг 2: Создать скилл `.cursor/skills/1c-vendor-standards/SKILL.md`

Формат: ~300-400 строк, как у существующих скиллов (description, trigger, instructions).

Структура:

- **Метаданные и именование** — чеклист для architect/reviewer
- **Хранение данных** — выбор объекта, проведение, активность движений
- **Обработчики событий** — ПередЗаписью/ПриЗаписи/ОбменДанными.Загрузка
- **Запросы** — ключевые правила (индексы, ПОДОБНО, соединения, ВТ, составные типы)
- **Транзакции и блокировки** — правила транзакций, ответственное чтение, разделение итогов
- **Код и модули** — структура, именование, переменные, исключения, экспорт
- **Клиент-сервер** — директивы, минимизация вызовов, ПовтИсп
- **Безопасность** — SQL injection, Выполнить/Вычислить, проверка прав, привилегированный режим
- **Формы и UI** — РеквизитФормыВЗначение, длительные операции, информирование
- **БСП и библиотеки** — переопределяемые модули, совместимость
- **Локализация** — НСтр, СтрШаблон

Каждый домен: 5-10 строк правил + ссылка на `standards-1c.md` (Grep по заголовку).
